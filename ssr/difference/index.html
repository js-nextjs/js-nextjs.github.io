    What is Server Component, React Server Component, Client Component

    Nice — let’s clarify the core concepts: Server Component, React Server Component (RSC), and Client Component — and how they work especially in Next.js App Router.

    🔄 Quick Summary Table

    Concept Runs On Can Use useState? Can Fetch Data? When to Use
    Server Component Server ❌ ✅ For data fetching + rendering
    React Server Component Server ❌ ✅ Fancy term for Server Component (RSC = Server Component)
    Client Component Browser ✅ ❌ (usually) For interactivity

    🧩 1. What is a React Server Component (RSC)?

    React Server Components (RSC) are a React feature that lets components run only on the server, and never sent to the browser as JS.

    ✔️ What they can do:
    • Fetch data from DB or API
    • Render static or dynamic HTML
    • Pass props to Client Components
    ❌ What they can’t do:
    • Use useState, useEffect, or window
    • Handle user interactions

    ✅ How to define:

    Just create a component without 'use client':

    // app/posts/page.tsx ← This is a Server Component
import { getPosts } from "@/lib/api";

export default async function PostsPage() {
  const posts = await getPosts();
  return (
    <div>
      {posts.map((post) => (
        <p key={post.id}>{post.title}</p>
      ))}
    </div>
  );
}

    In Next.js App Router, every component is a Server Component by default unless you opt-in to Client Component.

    🧠 2. What is a Server Component?

    It’s essentially the same thing as an RSC.

    • In the context of Next.js, Server Component = React Server Component.
    • It just means: this component executes on the server, not the browser.
    • It's part of React’s new rendering model that allows data-fetching right in your component with await.

    ✅ In Next.js, if you're in /app, and you don’t add 'use client', you’re writing a Server Component.

    🖱️ 3. What is a Client Component?

    Client Components run in the browser. You need them for:

    • Interactivity (onClick, onChange, etc.)
    • State and lifecycle hooks (useState, useEffect)
    • DOM APIs (window, document)
    • Forms, modals, buttons, animations, etc.

    ✅ How to define:

    Add 'use client' at the top:

    // app/components/Counter.tsx
"use client";

import { useState } from "react";

export default function Counter() {
  const [count, setCount] = useState(0);

  return <button onClick={() => setCount(count + 1)}>Count: {count}</button>;
}

    Then import into a Server Component:

    // app/page.tsx (Server Component)
import Counter from "./components/Counter";

export default function Home() {
  return (
    <main>
      <h1>Welcome</h1>
      <Counter /> {/* client component */}
    </main>
  );
}

    🔄 How Server & Client Components Work Together

    You use Server Components for:

    • Fetching data
    • Heavy rendering
    • Keeping client bundles small

    And Client Components for:

    • UI state
    • Event handling
    • Animations, modals, toggles
    // app/page.tsx
import Counter from "./components/Counter"; // client
import PostList from "./components/PostList"; // server

export default function Home() {
  return (
    <div>
      <PostList />
      <Counter />
    </div>
  );
}

    🧠 Remember:

    Feature Server Component (RSC) Client Component
    await fetch() ✅ ❌
    useState() ❌ ✅
    Runs on Server only Browser
    Bundle size impact Light (not shipped) Heavier (shipped)
    Interactivity (events etc) ❌ ✅
    'use client' needed? ❌ ✅
