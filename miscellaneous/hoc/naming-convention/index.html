    Naming Convention

    ‚úÖ Naming Conventions for HOCs and Hooks in React

    When working with HOCs and custom hooks, it's important to follow consistent naming conventions to improve code readability and maintainability.

    üìÑ Naming Conventions for Higher-Order Components (HOCs)

    ‚úÖ Filename:

    • Start the filename with with to indicate it's a HOC.

    ‚úÖ Example:

    Component Purpose Filename
    Authorization HOC withAuthorization.js
    Logging HOC withLogging.js
    Data Fetching HOC withDataFetching.js

    ‚úÖ Function Name:

    • Use the same name as the file for consistency.
    // File: withAuthorization.js
const withAuthorization = (WrappedComponent) => {
  return (props) => {
    const isAuthenticated = localStorage.getItem("token");
    return isAuthenticated ? (
      <WrappedComponent {...props} />
    ) : (
      <h1>Access Denied</h1>
    );
  };
};

export default withAuthorization;

    üìÑ Naming Conventions for Custom Hooks

    ‚úÖ Filename:

    • Start the filename with use to indicate it's a hook.

    ‚úÖ Example:

    Hook Purpose Filename
    Data Fetching Hook useFetchData.js
    Authentication Hook useAuth.js
    Debounce Hook useDebounce.js

    ‚úÖ Function Name:

    • Use the same name as the file for consistency.
    // File: useFetchData.js
import { useState, useEffect } from "react";

const useFetchData = (url) => {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    fetch(url)
      .then((response) => response.json())
      .then((data) => {
        setData(data);
        setLoading(false);
      });
  }, [url]);

  return { data, loading };
};

export default useFetchData;
